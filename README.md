# AI Habit Tracker

A habit-tracking application with AI-powered personalised tips, pattern analysis, and streak-risk detection. Built as a PoC for an ASDLC (AI-assisted Software Development Lifecycle) upskilling programme ‚Äî the codebase itself is produced by a swarm of BA, Architect, Developer, and Reviewer agents working under engineer supervision.

## What it does

- Create, list, update, and soft-delete habits via a REST API
- Daily personalised tips generated by Claude, augmented with your habit history via pgvector RAG
- Weekly pattern analysis and streak-risk alerts (upcoming)

## Prerequisites

- **JDK 11+**
- **Gradle 7+** (or use the wrapper: `backend/gradlew`)
- **Docker** (for PostgreSQL + pgvector)

## Running locally

### 1. Set environment variables

```bash
cp .env.example .env
# Edit .env ‚Äî at minimum set DB_PASSWORD and ANTHROPIC_API_KEY
```

### 2. Start the database

```bash
docker compose up -d
```

This starts PostgreSQL 17 with the pgvector extension on `localhost:5432`.
Wait for the health-check to go green (`docker compose ps`).

### 3. Run database migrations

```bash
cd backend
gradle flywayMigrate
```

### 4. Start the backend

```bash
cd backend
gradle run
```

The API is now available at `http://localhost:8080`.

## API quick reference

All endpoints are under `/habits`. Request and response bodies are JSON.

| Method | Path | Description | Success |
|--------|------|-------------|---------|
| `POST` | `/habits` | Create a habit | `201` |
| `GET` | `/habits` | List active habits | `200` |
| `GET` | `/habits/{id}` | Get a single habit | `200` |
| `PUT` | `/habits/{id}` | Update a habit (full replace) | `200` |
| `DELETE` | `/habits/{id}` | Soft-delete a habit | `204` |

**Create / update body:**
```json
{
  "name": "Read 20 pages",
  "description": "Non-fiction preferred",
  "frequency": "daily"
}
```
`frequency` accepts `"daily"` or `"weekly"`. `description` is optional.

**Error responses** always have the shape `{ "message": "..." }`.

## Running tests

Unit tests (no Docker required):

```bash

```

Integration tests (requires Docker):

```bash

```

Integration test classes are annotated `@Ignore` by default and must be run manually.
