## How to Start a Phase

### Full swarm run (all agents in sequence):
```bash
claude "
  Read docs/phases/phase_1_pattern_detection.md.
  Execute all agent roles in sequence: BA → Architect → Developer → Reviewer.
  Write each agent's output to the file path specified in the Agent Instructions section.
  Do not start the next role until the previous one is complete.
  Do not mark the phase done until all Done When checklist items pass.
"
```

### Individual agent run:
```bash
# BA only
claude "Read docs/phases/phase_1_pattern_detection.md.
Execute the BA agent role as described in the Agent Instructions section."

# Architect only (after BA has written PBI)
claude "Read docs/phases/phase_1_pattern_detection.md and
docs/phases/phase_1_pbi.md.
Execute the Architect agent role as described in the Agent Instructions section."

# Developer only (after Architect has written ADR)
claude "Read docs/phases/phase_1_pattern_detection.md,
docs/phases/phase_1_pbi.md, and
docs/adrs/ADR-001-phase1-pattern-detection.md.
Execute the Developer agent role as described in the Agent Instructions section."

# Reviewer only (after Developer is done)
claude "Read docs/phases/phase_1_pattern_detection.md and
docs/adrs/ADR-001-phase1-pattern-detection.md.
Execute the Reviewer agent role as described in the Agent Instructions section."
```

### Between-phase retrospective:
```bash
claude "Read docs/phases/retrospective.md and execute the Phase 1 retrospective."
```

## Phase Summary

### Phase 1 — Pattern Detection
SQL aggregations over habit data narrated by a direct Anthropic API call.
No RAG. Establishes: HabitContext, AnthropicClient, InsightPrompt pattern.
Gate: GET /users/{userId}/habits/insights returns 200 with analytics + narrative.

### Phase 2 — Habit Analysis
Richer SQL queries + PromptBuilder with named section methods.
No RAG. Extends: HabitContext with 3 new fields, introduces PromptBuilder.
Gate: GET /users/{userId}/habits/analysis returns 200 with extended analytics + narrative.

### Phase 3 — Basic RAG
pgvector + EmbeddingClient + TipRepository + static corpus.
Manual RAG pipeline with mandatory inline comments explaining each concept.
Gate: GET /users/{userId}/habits/tips returns 200 with retrieved tips + grounded narrative.

### Phase 4 — Full RAG Pipeline
Multi-source retrieval (tips + user notes) in parallel with Cats IO.
Deduplication, structured logging, eval endpoint.
Gate: Full pipeline working, all prior tests passing, eval endpoint operational.

## Hard Limits Applying to All Phases

These constraints apply throughout all phases and must be in every agent's
system prompt:

```
NO LangChain4j or any LLM framework
NO Future return types in route handlers or service methods
NO akka imports anywhere in src/
Model for developer agent LLM calls: claude-sonnet-4-20250514
API keys always from environment variables — never hardcoded
All prior phase tests must pass when a new phase is complete
```
